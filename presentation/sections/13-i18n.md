# 文件格式与 I/O

<div class="h-[2px] w-10 bg-sky-500 mt-2 mb-5"></div>

::left::

```json
{
  "version": 2,
  "canvas": { "width": 1200, "height": 800 },
  "shapes": [ ... ]
}
```

<div class="mt-4 text-sm text-slate-500">单个 shape 再细分为 `id / type / geometry / style / transform / zValue`，由 `shapeDataToJson()` 与 `shapeDataFromJson()` 统一编解码。</div>

::right::

<div class="space-y-4 text-sm">
  <div v-click class="rounded-xl border border-slate-200 p-4">
    <div class="font-semibold text-slate-700 mb-2">保存路径</div>
    <div class="text-slate-500">`FileManager::saveToFile()` 先检查 transform 是否受支持，再组装 `version + canvas + shapes` 根对象并写入缩进 JSON。</div>
  </div>
  <div v-click class="rounded-xl border border-slate-200 p-4">
    <div class="font-semibold text-slate-700 mb-2">加载路径</div>
    <div class="text-slate-500">`loadFromFile()` 会验证根对象、文档版本、单图形反序列化、transform 合法性和非空 ID 唯一性。</div>
  </div>
  <div v-click class="rounded-xl border border-sky-100 bg-sky-50 p-4 text-sky-700">
    当前文件格式只接受 <strong>version 2</strong>，并明确拒绝旋转 transform，换来更稳定、可测试的缩放与保存行为。
  </div>
</div>
